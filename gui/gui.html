For each symbolic ("formularesponse") problem in the course, submissions are grouped by numerical smapling (i.e., roughly by mathematical equivalence.)

  • Feedback Score (range: 0.0--1.0): Attempts to quantify how much a problem would benefit from wrong-answer feedback. A high score indicates that most wrong answers are of a few common types, so providing wrong-answer feedback is tractable.

  • \(n_{\text{correct}}\) (should be 1.0): The number of submission groups in whcih all submissions were graded as correct by edX. If \(n_{\text{correct}} \geq 2\), then multiple inequivalent answers are being graded correct by edX. This could indicate that the formularesponse grader is not strict enough.

  • \(n_{\text{partial}}\) (should be 0.0): the number of submission groups in which some but not all submissions were graded as correct by edX. This could indicate a few things:
    • The formularesponse grader was changed during the course run.
    • The formularesponse answer is randomized.
    • The formularesponse grader has is using bad samples. (EdX uses different random samples for each submission, so if sample range is bad the same submission may be graded different during each problem check.)
